# GodTodoList 需求文档

## 1. 项目概述

### 1.1 项目名称
GodTodoList — 目标驱动的高效事项管理体系

### 1.2 项目背景
企业研发环境中，工程师日常面临大量并行任务，缺乏将任务与个人目标关联、管理任务上下文和资料、追踪时间投入的统一工具。同时，企业内网环境限制使用外部 SaaS 服务，需要一个完全本地化运行的解决方案。

### 1.3 目标用户
- 企业研发工程师（主要）
- 需要管理多目标多任务的知识工作者
- Mac / Windows / Linux 多平台用户

### 1.4 设计原则
- **零外部依赖**：不依赖云服务、外部数据库，纯本地运行
- **零构建工具**：前端使用 Vue 3 CDN，无需 webpack/vite
- **零编译依赖**：数据库使用 sql.js（纯 JS SQLite），避免原生模块编译问题
- **跨平台兼容**：Mac、Windows、Linux 均可使用
- **数据可迁移**：支持目录导入导出，数据随文件夹走
- **安全优先**：路径验证、命令注入防护、本地绑定、设置白名单

## 3. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 性能 | 单用户场景，任务数 < 10000 时响应 < 200ms |
| NF-02 | 数据安全 | 所有数据本地存储，不联网传输（AI 除外） |
| NF-03 | 数据持久化 | SQLite 数据库文件存储，进程退出数据不丢失 |
| NF-04 | 可移植性 | 数据库文件 + 任务目录可直接拷贝到另一台机器使用 |
| NF-05 | 易用性 | 一键启动，零配置即可使用基本功能 |
| NF-06 | 中文优化 | 界面全中文，字体适配 PingFang SC / Microsoft YaHei |

## 4. 安全需求

| 编号 | 需求 | 说明 |
|------|------|------|
| S-01 | 路径安全 | 所有文件/目录操作验证路径在 root_dir 或任务 folder_path 下，拒绝路径遍历攻击 |
| S-02 | 命令注入防护 | 终端/OpenCode 启动使用 spawn + args 数组，禁止字符串拼接 exec |
| S-03 | 本地绑定 | 服务器仅监听 127.0.0.1，拒绝外部网络访问 |
| S-04 | 设置白名单 | PUT /api/settings 仅接受已知 key 列表，防止注入任意设置项 |
| S-05 | 附件删除验证 | 删除附件前验证文件路径在对应任务的 folder_path 下 |
| S-06 | 扫描范围限制 | 目录扫描（导入/文件发现）限制在 root_dir 及其子目录内 |
| S-07 | 统一错误处理 | asyncHandler 中间件包装所有路由，异常统一返回 500 + toast 提示 |
| S-08 | 前端错误捕获 | api() 函数全局 catch，网络异常时 toast 提示不阻塞 UI |

## 5. 约束条件

- Node.js 18+ 运行环境
- 不使用任何需要编译的原生模块
- AI 功能需要用户自行配置 API Key
- 文件管理功能需要配置根目录路径
