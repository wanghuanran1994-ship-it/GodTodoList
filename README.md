# GodTodoList

> 目标驱动的高效事项管理体系 — 全栈、本地化、AI 增强

## 背景

企业研发环境中，工程师日常面临大量并行任务：
- 任务零散、缺乏与个人目标的关联
- 汇报任务多且重要，占时间、影响影响力，却缺乏专门管理
- 任务上下文（资料、对话）分散在不同工具中
- AI 编码助手（Claude Code / OpenCode）的对话无法关联到具体任务
- 内网环境限制使用外部 SaaS，需要一个完全本地化的解决方案

GodTodoList 解决这些问题：以目标为驱动、以任务为中心、AI 为辅助，将所有工作上下文聚合在一起。

## 一键启动

```bash
npm install && npm start
```

浏览器自动打开 `http://localhost:3000`。

Windows 用户也可双击 `start.bat`，Mac/Linux 用户双击 `start.sh`。

## 设计原则

- **零外部依赖** — 不依赖云服务、外部数据库，纯本地运行
- **零构建工具** — 前端 Vue 3 CDN，无需 webpack/vite
- **零编译依赖** — 数据库使用 sql.js（纯 JS SQLite），避免原生模块编译
- **跨平台** — Mac / Windows / Linux 均可使用
- **数据可迁移** — 数据库文件 + 任务目录可直接拷贝到另一台机器
- **安全优先** — 路径验证、命令注入防护、仅监听 127.0.0.1

## 功能

### 任务管理
- **看板视图** — 3 列拖拽（正在进行 / 搁置 / 已完成），状态灯区分：灰色=未开始、绿色=进行中、黄色=搁置、深灰=已完成
- **快速创建** — 一行自然语言输入，支持 `@明天 #标签 +人名 !30m 🔥` 语法，AI 自动解析
- **批量操作** — 多选批量改状态、改目标、标记今日、删除
- **子任务** — 任务下多级子任务，进度百分比展示，支持 AI 自动分解
- **父子任务** — 任务可设置父任务，卡片显示层级关系徽章
- **截止日期** — 逾期自动高亮提醒
- **计时器** — 任务级计时，记录实际投入时间
- **时间日志** — 多条时间记录，预估/实际对比
- **标签系统** — 双维度标签（紧迫度 + 价值），支持自定义
- **相关人员** — 任务关联人员管理
- **任务复制** — 一键复制任务元数据

### 工作台
- 概览卡片：待办 / 进行中 / 已完成 / 已逾期数量
- 今日任务列表（时间预算 8h）
- 即将到期任务
- 最近完成任务

### 汇报管理
- 独立汇报视图，按会议分组展示（部门周会、月度 Review 等）
- 任务标记为汇报，关联到具体会议
- 看板筛选栏一键筛选汇报任务
- 惯例生成任务时自动继承汇报属性
- 已有会议名自动补全

### 日历
- 月视图：日期格子 + 任务状态点
- 周视图：7 列展示，每列显示当日任务详情
- 点击任务跳转详情

### 目标管理
- 目标卡片：名称、描述、颜色
- 进度条（完成率、任务数统计）
- 距截止日天数、预估/实际时间对比
- 目标关联路径列表，一键打开

### 惯例系统
- 周期性任务模板：每天 / 每周 / 每两周 / 每月 / 每季度
- 从惯例一键生成任务（继承目标、标签、汇报属性）
- 支持归档

### AI 军师
- 浮动对话面板，可随时唤起
- 兼容 OpenAI 协议（DeepSeek / GLM / 通义 / Ollama）
- 兼容 Anthropic 协议（智谱等）
- SSE 流式响应，实时显示
- 自动注入任务上下文（目标、活跃任务、今日完成）
- 快捷提示：今日规划 / 目标分析 / 周报生成
- AI 任务丰富：自动建议描述、时间、标签、子任务
- AI 自动生成日报/周报（流式、可编辑、可复制）

### AI 会话关联
- 扫描 Claude Code 会话（`~/.claude/projects/`）关联到任务
- 扫描 OpenCode 会话（`~/.local/share/opencode/storage/`）关联到任务
- 一键在终端恢复对话（`claude --resume` / `opencode --session`）
- 任务详情面板展示关联会话列表

### 文件管理
- 任务创建时自动创建文件夹（日期_名称 / 编号_名称格式）
- 拖拽文件上传，自动归档到任务目录
- 自动生成 README.md（YAML frontmatter + 描述 + 背景）
- 任务更新时自动同步 README.md
- 任务目录文件自动发现
- 关联路径管理（主路径 + 附加路径，可拖拽排序）
- 一键打开文件夹 / 终端 / 编辑器

### 批量导入
- 扫描目录识别子文件夹
- 解析 README.md YAML frontmatter
- 预览列表，勾选导入
- 已有任务自动去重

### 笔记卡片
- 便签式笔记系统，支持多卡片
- 每个卡片支持多条条目（类 checklist）
- 卡片重命名、删除

### 统计 & 回顾
- 时间统计：日/周/月趋势图
- 目标进度统计
- 日报/周报回顾（AI 自动生成 + 可编辑）

### 其他
- 深色模式
- 侧边栏可折叠
- 数据备份：一键下载/恢复数据库
- 快捷键：Ctrl+N 新建任务、Space 开始/停止计时、T 标记今日、←→ 切换任务、1-4 改状态

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | Vue 3 (CDN) + 原生 CSS | Composition API，单文件 SPA |
| 后端 | Express | RESTful API + SSE 流式 |
| 数据库 | sql.js (SQLite WASM) | 零编译、单文件、跨平台 |
| 文件上传 | Multer | Express 标准中间件 |
| AI 通信 | HTTP + SSE | 兼容 OpenAI / Anthropic 协议 |

## 项目结构

```
GodTodoList/
├── server/
│   ├── index.js          # Express 服务、路由、AI 代理、安全中间件
│   ├── database.js       # SQLite 数据访问层
│   └── fileManager.js    # 文件操作、README 读写、目录扫描
├── public/
│   ├── index.html        # SPA 页面结构
│   ├── css/style.css     # 全局样式
│   └── js/app.js         # Vue 3 应用逻辑
├── data/                 # 运行时数据（gitignored）
├── docs/                 # 设计文档
├── start.sh / start.bat  # 启动脚本
└── package.json
```

## 安全

- 仅监听 `127.0.0.1`，拒绝外部网络访问
- 所有文件操作验证路径在允许范围内，拒绝路径遍历
- 终端/编辑器启动使用 `spawn` + args 数组，防止命令注入
- 设置项白名单校验
- 统一错误处理，异常不泄漏敏感信息

## 约束

- Node.js 18+
- AI 功能需自行配置 API Key
- 文件管理需在设置中配置根目录路径
